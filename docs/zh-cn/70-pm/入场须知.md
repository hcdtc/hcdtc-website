# 入场须知

例如：
```
关于前期的准备工作及现场的管理工作，包括但不限于如下关键事项：
 
项目概况：
本项目将基于HPM平台改造，（目前此平台没有技术团队在持续维护）因此，重点是前期需要梳理需求，理清HPM标准功能与客户具体需求之间的匹配程度，异同点，把握标准功能改造及客户化需求的边界，尽量不要重复造轮子；关于HPM技术问题，可找冉龙梅帮忙协助解决。
 
前期成果：
售前投标文件-技术卷：如附件，基本涵盖了客户对技术的主要关注点，例如，服务对接、数据安全性考虑等等
物理部署架构：与客户讨论过一版，如附件PPT，大部分组件客户基础设施建设已满足；需要说明的是，客户希望使用MySQL，具体说明见附件
初版开发清单：还是比较粗的一版，需要现场你与建强一起参照客户化需求，并结合HPM标准功能匹配程度进行完成，含改造及客户化，形成范围基准
 
准备工作：
搭建SVN统一管理成果物
与项目经理一起制定项目规范，并在技术团队内宣贯；
制定统一开发规范（需要考虑是否与HPM之前的规范保持兼容，正常将应该不会存在较大偏差）
梳理技术路线，标识新技术、难点技术等，例如，H5页面开发、ERP相关接口准备（Oracle团队应该会有），提前安排团队成员研究攻坚
 
范围与计划：
与项目经理一起梳理客户需求，尽快确定初版开发清单并邮件发布，涵盖HPM改造及个性化需求，尽量不要遗漏
制定总体开发计划并邮件发布，务必与项目经理一起确认关键里程碑节点
每周制定双周滚动计划并邮件做简短工作总结，尤其注意风险识别、跟踪及确认
 
质量管理：
每周固定时间代码检查（静态代码规范可通过CheckStyle等工具自动检测），尤其是新人；不定期组织分享，不断提高团队成员开发能力和质量
核心设计如把握不准可以找我或者组长进行研讨确认
 
其他：
设计与开发尽量考虑通用性、扩展性，后续新希望的另外一个项目极有可能也是采用HPM+客户化的方式进行，这点与项目经理多沟通
 
其他后续进行中再逐步沟通。
 
Thanks & Best Regards
==================
刘明伟 Allen Liu
```
